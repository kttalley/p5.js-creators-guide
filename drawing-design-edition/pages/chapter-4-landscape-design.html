Designing an Illustration in Advanced

Although adjusting to the larger, to-scale canvas size of Advanced can be overwhelming, that's not the only thing we must be comfortable working with. There are two new functions provided: function setup ( ) { } and function draw ( ) { }. In order to understand how function setup and function draw are working, we have to understand that our browsers, the software we use to request and view different web pages, interprets or reads our code line by line, in the order the lines of code are written.

Even though it may look as if our computer is executing our code in a single instant, JavaScript is actually interpreting our code from the first line that's written until the last. This is why when we begin illustrating with shape functions, we notice that the final shape function called on the last line of our code is always the top-most visible layer on our sketch's canvas. It's as if every line of code calling a shape function is telling your computer to cut that shape out of construction paper and glue it down on top of your canvas in the place you described, regardless of whether or not something has already been drawn in that area. This is also why if you include the background( ) color function on the last line of your project's code, then the very last thing your computer would interpret and execute from your code is an opaque background on top of all the visuals that are drawn before that final step. We must always be careful and intentional with the order of our instructions to the computer, especially when creating visual projects.

Being able to slow down and break down each step the computer is carrying out as a result of our code is a very important skill that you will naturally develop over time. Thankfully, programming visual projects makes it easier to form this understanding of how and when our lines of JavaScript code are being executed. function setup ( ) and function draw allow us to control the flow or order of our instructions to the computer even further.
